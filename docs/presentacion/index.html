Sports Bar

API para gestionar peñas de fútbol

TFG realizado por Manuel Jurado Trillo

Problema encontrado

  • Pérdida de tiempo en creación de equipos
  • Partido igualado -> Más diversión

Estado del arte

Creación de equipos

Aplicaciones que reparten jugadores en equipos pero de manera aleatoria

Gestión de niveles por jugador

Aplicaciones que asignan un nivel a un jugador en función de su rendimiento

Mi propuesta

Creación de una API REST que cumpliera los siguiente objetivos

  1. Creación de equipos igualados en función del nivel
  2. Gestión del nivel en función de los resultados

Metodología

  • SCRUM adaptado
    • Entregas parciales y regulares (PMV)
    • Convertir historias de usuario en una serie de hitos

Planificación

Control de calidad

Análisis del problema

Identificación y estructura de datos de los objetos valor

  • Nivel
  • Amigo
  • Equipo

Identificación y estructura de datos de la entidad

  • Grupo de amigos

Creación de algoritmo para creación de equipos igualados

  • Ordenar la lista de jugadores disponible en nivel decreciente
  • Asignar el siguiente jugador de la lista al equipo con menos nivel acumulado
  • Algoritmo válido si mejora la aleatoriedad

Gestion de niveles post-partido

  • Aumentan un nivel si ganan el partido
  • Pierden un nivel si pierden el partido
  • Siempre respetando el rango de valores del tipo de dato Nivel
