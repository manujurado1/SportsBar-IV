Sports Bar

Backend para gestionar peñas de fútbol

TFG realizado por Manuel Jurado Trillo

Problema encontrado

Buscar problemas que se me presentaran en el día a día.

Cada vez que me reunía con amigos/compañeros de trabajo para jugar a fútbol pasaba lo siguiente:

  • Pérdida de tiempo en creación de equipos
  • Partido igualado → Más diversión

Estado del arte

  • Creación de equipos igualados
  • Gestión de nivel por jugador en función de su rendimiento

Creación de equipos

Aplicaciones que reparten jugadores en equipos pero de manera aleatoria.

Gestión de niveles por jugador

Aplicaciones que asignan un nivel a un jugador en función de su rendimiento.

Metodología

  • Desarrollo ágil
    • Entregas parciales y regulares (PMV)
      • "Nuestra mayor prioridad es satisfacer al cliente mediante la entrega temprana y continua de software con valor."
    • Convertir historias de usuario en una serie de hitos

Planificación

Historias de usuario → Issues → Hitos (PMV cada vez más complejos)

Este proceso se ha llevado a cabo siguiendo el Diseño Dirigido por Dominio (DDD).

Seguimiento del desarrollo

  • Control de versiones : Git
  • Alojado en GitHub

Control de calidad

"La atención continua a la excelencia técnica y al buen diseño mejora la Agilidad."
  • Test siguiendo TDD
    • "La simplicidad, o el arte de maximizar la cantidad de trabajo no realizado, es esencial."
  • Linter: Golangci-lint
  • GitHub Actions y Checks API

Mi propuesta

Creación de una API REST que cumpliera los siguiente objetivos:

  1. Creación de equipos igualados en función del nivel
  2. Gestión del nivel en función de los resultados

Creación de algoritmo para creación de equipos igualados

  • Ordenar la lista de jugadores disponible en nivel decreciente
  • Asignar el siguiente jugador de la lista al equipo con menos nivel acumulado
  • Algoritmo válido si mejora la aleatoriedad

Gestión de niveles post-partido

  • Aumentan un nivel si ganan el partido
  • Pierden un nivel si pierden el partido
  • Siempre respetando el rango de valores del tipo de dato Nivel

3. Creación API REST

  • Conveniencia de usar framework

Conveniencia de usar framework

Go tiene su biblioteca estándar, por lo que es necesario saber si merece la pena añadir una capa más al proyecto usando algún framework.

Se ha decidido utilizar un framework para la creación de la API debido a:

  • Gracias a Go Modules no es un problema añadir dependencias al proyecto
  • Facilidades a la hora de crear la API y manejar las solicitudes y los errores

Elección del framework

Criterios:

  • Freshness
